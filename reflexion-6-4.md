## Reflexión

1. Diferencia:
git merge crea un commit de fusión y mantiene el historial con ramas.
git rebase reescribe el historial para que sea lineal.

2. Cuándo usar:
Usaría rebase en ramas locales para mantener el historial limpio.
Usaría merge cuando trabajo en equipo.

3. Por qué no usar rebase en remoto:
Porque modifica el historial y puede causar problemas a otros desarrolladores.
